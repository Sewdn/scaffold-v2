## What is Scaffold CLI?

Scaffold CLI v2 is a **command-orchestration** tool with a layered approach. Template files (Mustache stubs) are copied and rendered; additional commands (bun create, bun add, etc.) extend the generated code. Dependency management for TypeScript packages is consolidated and centralized.

## Key features

<CardGrid stagger>
  <Card title="Create Projects" icon="file">
    Bootstrap full monorepos with `scaffold project` or incremental structures with `scaffold init`.
  </Card>
  <Card title="Add Apps" icon="add-document">
    Add Next.js, Vite, backend (Elysia), CLI, MCP server, slides, and docs apps.
  </Card>
  <Card title="Add Packages" icon="pencil">
    Create service packages, UI packages, and modules (service + UI pairs).
  </Card>
  <Card title="AI-Ready" icon="setting">
    Stable CLI API documented for AI agents and MCP integration.
  </Card>
</CardGrid>

## Get started

```bash
# Create a monorepo with apps
bun run scaffold project my-app --apps frontend-vite,backend --app-names web,api --non-interactive

# Or initialize base structure and add apps manually
bun run scaffold init my-app
cd my-app
bun run scaffold app web --type frontend-vite --non-interactive
```
