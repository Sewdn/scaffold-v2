# Success Metrics

| Metric | Target |
|--------|--------|
| Expansion commands implemented | 60+ |
| App types with expansion | 20+ |
| Service packages with expansion | 17+ |
| E2E scenarios for expansion | 15+ |
| Determinism | Same command → identical output |
| AI agent token reduction | Measurable via fewer structural prompts |

## Technology Alignment

- **Effect** adoption: % of expanded services/hooks using Effect. **Registry** coverage: all expansions patch registries.
- **Implementation implication:** Metrics track expansion count, Effect usage, determinism. Alternatives: prompt-length, manual edits.
- **Alternative philosophy:** Prompt-heavy metrics (token count) vs structure-heavy (expansion count, determinism) — we track both.
